========================================================================
  一、项目概述
========================================================================

【产品定位】
  一个功能齐全、设计精美的免费在线节拍器网站。
  对标产品：metronome-online.com / online-metronome.org

【核心卖点】
  - 打开即用，无需下载
  - 界面比竞品更好看、更现代
  - 功能比基础版竞品更多（tap tempo、subdivisions、trainer mode）

【盈利模式】
  100% 靠广告 —— Google AdSense 为主要收入来源
  后期可加：乐器配件 affiliate 链接（吉他调音器、谱架等）

【为什么这个赛道能赚钱】
  - 搜索量：全球每月 "online metronome" 搜索量约 50-80 万次
  - 竞品分析：online-metronome.org 月流量约 200 万访问
  - 广告单价：音乐类 CPM 约 $2-8
  - 简单计算：如果做到月访问 10 万 → 月广告收入约 $500-2000

========================================================================
  二、竞品分析
========================================================================

对标产品（按流量从大到小）：

  1. online-metronome.org — 月流量 ~200 万
     优势：SEO 历史悠久、Google 排名第一
     劣势：界面老旧、功能简单、广告太密集

  2. metronome-online.com — 月流量 ~50 万
     优势：支持多语言（含中文）、界面中等
     劣势：广告位置太多影响体验

  3. itselftools.com/metronome — 月流量 ~20 万
     优势：功能最强（swing、trainer、presets）
     劣势：界面偏极客，普通用户学习成本高

  4. 吉他调音器类App的内置节拍器 (GuitarTuna, Tunable)
     优势：有 App 用户基础
     劣势：网页版是副业，不是主产品

【市场机会】
  - 大部分竞品界面停留在 2015 年设计水平
  - 移动端体验普遍不好
  - 没有一个竞品做多语言（特别是中文）
  - "漂亮 + 好用 + 中文" 三个词在节拍器赛道目前是真空

========================================================================
  五、功能规划
========================================================================

【Phase 1 — MVP 最小可行产品】3-5 天

  1. BPM 主控
     - 大号数字显示当前 BPM
     - 滑块调节 (20-300 BPM)
     - +/- 按钮微调 (1 BPM 步进)
     - 键盘快捷键 (上下箭头)

  2. 拍子/节奏
     - 每小节拍数选择 (1-12 beats)
     - 常用预设：2/4, 3/4, 4/4, 6/8
     - 每拍可视化指示灯（播放时跳动）

  3. 播放控制
     - 播放/暂停按钮（空格键）
     - Tap Tempo（手点测速）

  4. 视觉设计
     - 深色主题（多数音乐人偏好）
     - 移动端适配
     - 大号可触摸按钮

  5. 广告位
     - 顶部横幅 (728×90)
     - 右侧方框 (300×250)
     - 移动端自适应调整

【Phase 2 — 增强功能】+3 天

  6. 拍子细分 (Subdivisions)
     - 四分音符 / 八分音符 / 三连音 / 十六分音符
     - 每拍多个细分声音

  7. 音量控制
     - 强拍音量 / 弱拍音量 / 细分音量 独立调节

  8. 音色选择
     - 至少 3 种：经典电子音 / 木鱼声 / 鼓掌声
     - 预设快速切换

  9. 速度预设
     - Largo / Adagio / Andante / Moderato / Allegro / Presto
     - 一键切换

【Phase 3 — 差异化功能】+5 天

  10. 训练模式
      - 倒计时：播放 N 小节 → 静音 N 小节 → 检查是否还跟得上
      - 渐进加速：从 A BPM 自动逐步加到 B BPM

  11. 多语言
      - 中文 / 英文 / 日文 / 西语 / 葡语
      - 自动检测浏览器语言

  12. 节拍器历史/预设保存
      - localStorage 保存常用设置
      - 一键恢复上次设置

  13. 分享功能
      - 生成带参数的 URL (如 ?bpm=120&beat=4)
      - 复制链接分享给朋友

========================================================================
  六、技术选型
========================================================================

  技术              用途                    原因
  ─────────────────────────────────────────────────────
  HTML5              结构                  纯静态，加载快
  Tailwind CSS       样式                  CDN 引入，开发快
  Web Audio API      音频                  sample-accurate 精度
                      (不用 setTimeout)     不漂移，专业级
  Vanilla JS         逻辑                  零依赖，页面超轻
  Google AdSense     广告                  插入即用
  Vercel/GitHub      部署                  免费托管

  为什么不用 App？
  - 网页 = 打开即用，不需要下载
  - SEO 免费流量
  - 开发和维护成本低

========================================================================
  十、风险与对策
========================================================================

  风险 1：Google AdSense 审核不通过
  对策：先上线，积累 1-2 周真实流量后再申请；确保有隐私政策页面

  风险 2：竞品太强，SEO 排名上不去
  对策：做长尾关键词 + 多语言，不在主词上和 online-metronome.org 硬拼

  风险 3：广告收入太低
  对策：后期加 affiliate 链接（乐器电商、在线课程）

  风险 4：Web Audio API 兼容性问题
  对策：加 fallback（AudioContext 降级到 webkitAudioContext）
